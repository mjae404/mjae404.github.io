Git & GitHub 개요

2020-08-27
git

현대 개발 팀 프로젝트에서 버전관리와 클라우드 저장소는 꼭 필요한 것이며, 버전관리를 위해서 Git, 클라우드 저장을 위해서 Github를 익히면 팀프로젝트 때 효율적인 개발 협업을 할 수 있다.

다수의 개발자가 하나의 서비스를 만드려면?

  1. 각자 개발하다가 메일이나 USB 등으로 보내서 합친다.
  2. 에러를 대비해 백업본을 만들어둔다.
  3. 상대방의 파일을 수정시에 따로 언급한다.
  4. 코드를 합칠 때 내 코드에도 반영한다.

→ 따로 작업하다가 내가 원할 때 코드를 합칠 수 있는 방법: 깃 (분산 버전 관리 시스템)

버전 관리 시스템 Git

내가 만든 버전과 동료가 만든 버전을 비교해서 최신본으로 코드를 업데이트할 수 있다. 저장할 공간만 있다면 어디서나 사용이 가능하다. CLI(Command-line interface), GUI(graphical user interface)를 사용할 수 있다.

Github에 코드를 올리는 과정

  1. 해당 프로젝트 폴더에서 사용할 것이라고 명령
  2. 작업 후 변경한 파일중 올리고 싶은 것을 선택
  3. 선택한 파일을 하나의 버전으로 만들어주고 설명을 기재
  4. GitHub 사이트에서 프로젝트 저장소 생성
  5. 내 컴퓨터에서 만들었던 덩어리 작업물 GitHub에 업데이트
