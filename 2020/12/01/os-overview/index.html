운영체제 개요

2020-12-01

운영체제 역할

  1. 시스템 자원(System Resource) 관리자
  2. 사용과 컴퓨터간의 커뮤니케이션 지원
  3. 컴퓨터 하드웨어와 응용 프로그램을 제어

컴퓨터 하드웨어는 스스로 할 수 있는 것이 없다. 그래서 운영체제가 필요하다.

  1. CPU: 각 프로그램이 얼마나 CPU를 사용할지를 결정할 수는 없다.
  2. Memory: 각 프로그램이 어떤 주소에 저장되어야 하는지, 어느 정도의 메모리 공간을 확보해주어야 하는지 결정할 수는 없다.
  3. 저장매체(HDD, SSD): 어떻게, 어디에 저장할지 결정할 수 없다.
  4. 키보드/마우스: 스스로 표시할 수 없다.
  • 운영체제, Operating System 또는 OS
  • 시스템 자원(System Resource) = 컴퓨터 하드웨어
    • CPU(중앙처리장치), Memory(DRAM, RAM)
    • I/O Devices(입출력장치)
      • Monitor, Mouse,Keyboard, Network
      • 저장매체: SSD, HDD(하드디스크)

대표적인 운영체제

  • Windows OS
  • Mac OS
  • UNIX(유닉스)
    • UNIX 계열 OS: LINUX(리눅스) OS
      • UNIX와 사용법이나 OS 구조가 유사
    • LINUX(리눅스) OS
      • 프로그래머나 전공자는 필수적으로 알고 있어야 한다.
      • 서버 환경에서 주로 작업 및 사용한다.

응용 프로그램

  • 프로그램 = 소프트웨어
  • 소프트웨어 = 운영체제, 응용 프로그램(엑셀, 파워포인트 etc)
  • 응용 프로그램 = Application = App
    • Application: 일반 PC에서의 응용 프로그램
    • App: 스마트폰에서의 응용 프로그램

운영체제와 응용프로그램간의 관계

  • 응용 프로그램을 관리한다
    • 응용프로그램을 실행
    • 응용 프로그램 간의 권한을 관리 (ex. 관리자 권한 실행)
    • 응용프로그램 사용자 관리 (ex. 로그인)
  • 응용 프로그램은 누구나 만들 수 있기 때문에 생기는 문제, 비정상적으로 작동하는 문제를 막거나 해결
    • 예시1. 무한 반복문을 넣었을 경우 끝나지 않고 계속 컴퓨터 자원을 잡아먹으며 실행되는 경우
    • 예시2. 응용 프로그램을 잘못 작성했을 시에 프로그램이 다운되는 경우
    • 예시3. 모든 파일 삭제하라는 코드를 삽입했을 경우 (권한/사용자 관리)
    • 예시4. 응용 프로그램이 CPU 전체를 강제적으로 사용하려고 시도하는 경우

운영체제의 목표
사용자가 사용하는 응용 프로그램이 효율적으로 적절하게 동작하도록 지원한다.

운영체제
응용 프로그램이 요청하는 시스템 리소스를 효율적으로 분배하고 지원하는 소프트웨어

컴퓨터 구조

컴퓨터를 키면 운영체제는 Memory에 올라가게 된다.
(참고: 폰노이만 구조)
