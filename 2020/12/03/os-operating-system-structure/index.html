운영체제 구조

2020-12-03
Computer Science / Operating System

응용 프로그램, 운영체제, 컴퓨터 하드웨어(시스템 리소스)의 관계

  • 운영 체제는 응용 프로그램이 요청하는 메모리를 허가하고 분배한다.
  • 운영 체제는 응용 프로그램이 요청하는 CPU 시간을 제공한다.
  • 운영 체제는 응용 프로그램이 요청하는 IO Devices 사용을 허가/제어한다.

운영체제: 도서관 / 응용프로그램: 시민 / 컴퓨터 하드웨어: 책
시민이 도서관에 원하는 책(자원)을 요청하면 도서관(운영체제)은 적절한 책(자원)을 찾아서 빌려준다. 기한이 지나면 도서관(운영체제)는 해당 책(자원)을 회수한다.

운영체제가 제공하는 인터페이스

운영체제는 컴퓨터 하드웨어와 응용 프로그램을 관리한다. 사용자 인터페이스를 제공하기 위해 Shell 프로그램을, 응용프로그램이 운영체제 기능을 요청하기 위해 시스템 콜을 제공한다. 시스템 콜을 직접 사용할 수도 있지만, 보통 직접 사용하기보다는 사용자가 운영체제에게 Shell을 통해 요청하거나, Application이 운영체제에게 요청할 때 API(함수의 집합)를 통해 요청 한다.

응용 프로그램은 운영체제 기능 필요시 해당 API를 사용하여 프로그램을 작성한다. 응용 프로그램이 실행되어 운영체제 기능이 필요한 API를 호출하면 시스템 콜이 호출되는데, 이 때 커널 모드로 변경되어 OS 내부에서 해당 명령이 실행 된다. 그 후에 다시 응용 프로그램으로 돌아간다.

사용자를 위한 인터페이스, 쉘(Shell)

  • 사용자가 운영체제 기능과 서비스를 조작할 수 있도록 인터페이스를 제공하는 응용 프로그램
  • 쉘은 터미널 환경(CLI)과 GUI 환경 두가지로 분류된다.

응용 프로그램을 위한 인터페이스

  • API (Application Programming Interface): 각 언어별 운영체제 기능 호출 인터페이스 함수
    • 함수로 제공 (함수의 집합)
    • 시스템콜을 감싸고 있다.
    • open() …
  • 라이브러리(library)의 형태
    • C library(glibc)
  • 시스템 콜: 운영체제 기능을 호출하는 함수
    • 시스템 콜 또는 시스템 호출 인터페이스 라고 한다.
    • 운영체제가 운영체제 각 기능을 사용할 수 있도록 시스템 콜이라는 명령 또는 함수를 제공한다.
    • API 내부에는 시스템 콜을 호출하는 형태로 만들어지는 경우가 대부분이다.
    • 시스템콜의 예
      • POSIX API(맥 OS, 유닉스 계열의 운영체제들이 주로 따르고 있는 시스템 콜 정의)
      • 윈도우 API

운영체제를 만든다면?

  1. 운영체제 개발 (kernel, 커널)
  2. 시스템 콜을 개발
  3. API(library) 개발
  4. Shell 프로그램 개발
  5. 응용 프로그램 개발

CPU Protection Rings

CPU의 두가지 모드

  • 사용자 모드 (user mode by applications): 응용 프로그램, API를 조작한다.
  • 커널 모드 (kernel mode by OS): 특권 명령어 실행과 원하는 작업 수행을 위한 자원 접근을 가능케 하는 모드
    • 커널(Level 0) - OS 서비스(Level 1, 2) - 응용 프로그램(Level 3)
    • OS 안으로 들어가 메모리, 디스크, 네트워크를 조작한다. 커널 모드에서만 실행 가능한 기능들이 있다.
    • 커널 모드로 실행하려면, 반드시 시스템 콜을 사용하여야 한다.(거쳐야 한다)
    • 시스템 콜은 운영체제 제공한다.
    • 함부로 응용 프로그램이 전체 컴퓨터 시스템을 해치지 못하게 한다.
