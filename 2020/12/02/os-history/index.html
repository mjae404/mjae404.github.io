운영체제 역사와 핵심기술

2020-12-02
  • 1950년대: 제일 처음 만들어진 컴퓨터, ENIAC
  • 1960년대: 배치 처리 시스템
  • 1970년대: 시분할 시스템/멀티태스킹 시스템과 UNIX OS
  • 1980년대: GUI, 개인용 컴퓨터
  • 1990년대: 다양한 응용프로그램 및 인터넷 발달, 오픈
  • 2000년대: 오픈소스 활성화, 가상머신, 대용량 병렬처리

1950년대

  • ENIAC: 첫번째 컴퓨터
  • 운영체제가 없음
    • 한번에 하나의 응용 프로그램을 실행
    • 응용 프로그램이 시스템 자원을 제어

1960년대 초반

  • 프로그램 종류도 많아지고 사용자도 많아지기 시작
  • 배치 처리 시스템 (batch processing system) 출현
    • 여러 응용 프로그램을 순서를 포함하여 등록시켜두면 자동으로 순차적으로 실행하는 시스템
    • 배치 처리 시스템을 기반으로 운영체제가 출현
    • 컴퓨터 응답 시간(response time)이 오래 걸릴 수 있다.
    • CPU가 필요 없음에도 프로그램이 CPU를 점유할 수 있기 때문에 실행 시간이 오래 걸릴 수 있다.

배치 처리 시스템

프로그램이 각각 걸리는 시간이 12시간, 1시간일 경우 프로그램1을 먼저 실행시키고 프로그램2를 실행시킨다면 프로그램1이 걸리는 시간은 12시간, 프로그램2가 걸리는 시간은 13시간이다. 프로그램2를 먼저 실행시키면 프로그램2의 실행시간은 1시간, 프로그램1의 실행시간은 13시간이다.

1960년대 후반

시분할 시스템 / 멀티 태스킹(멀티 프로그래밍) 이라는 새로운 개념이 제안되었다. 운영체제로 구현되지는 않았다. 시분할 시스템과 멀티태스킹을 동일하게 보는 시각도 있다. 결과적으로 핵심 기술은 시간을 잘게 쪼개 여러 응용 프로그램 실행한다는 것 이다.

  • 시분할 시스템(Time Sharing System)
    • 응용 프로그램이 CPU를 사용하는 시간을 쪼개 여러개의 응용 프로그램을 동시에 실행.
    • 다중 사용자를 지원하고, 컴퓨터 응답시간을 최소화하는 시스템.
  • 멀티 태스킹 (Multi Tasking)
    • 단일 CPU에서 여러 응용 프로그램의 병렬 실행을 가능케 하는 시스템.
    • 결과적으로 사용자가 느낄 때에는 여러 응용 프로그램이 동시에 실행되는 것처럼 보인다.
  • 멀티 프로그래밍
    • 가능한 CPU를 많이 활용하도록 하는 시스템
    • 시간 대비 CPU 사용률↑
    • 전체 응용 프로그램의 실행 시간을 줄일 수 있다.
    • A프로그램을 읽어들이는 동안 B프로그램을 실행하고, A프로그램을 읽는 것을 완료했다면 B프로그램을 멈추고 A프로그램을 실행하기 시작한다.

컴퓨터 응답 시간
A, B 프로그램이 순차적으로 진행되고 있다. A 프로그램은 3시간이 걸리고 B 프로그램은 키보드를 눌렀을 때 화면에 출력하는 프로그램이라면, A 프로그램이 실행중에 키보드를 눌렀다고 하더라도 A 프로그램이 실행 완료되어야(3시간) 화면에 출력된다. 이때 컴퓨터가 반응하는 시간(화면에 출력되기까지 걸리는 시간)을 컴퓨터 응답 시간 이라고 한다.

1970대

본격적으로 운영체제 중요성이 부각되어 현대 운영체제 기술이 확립되었다. 제대로 된 운영체제인 UNIX OS가 최초로 구현되었다.

UNIX(유닉스) OS 특징

  • 미국 AT&T사의 벨 연구소
  • 켄톰슨, 데니스 리치(C언어 개발)
  • 현대 운영체제의 기본 기술을 모두 포함한 최초의 운영체제
  • 멀티 태스킹, 시분할 시스템, 멀티 프로그래밍, 다중 사용자 지원

운영체제 개발에 C언어가 필요했던 이유

  • 1970년대 이전: Assembly 언어로 개발
    • CPU(명령어), Memory(물리적 주소)를 직접 작성
    • 컴퓨터마다 CPU나 Memory 등이 다를 수 있기 때문에 각각 다시 개발해주어야 하는 단점이 있다.
    • 프로그래밍 복잡도 높음.
  • 1970년대 이후: C언어가 개발됨
    • 컴파일러: 다시 작성할 필요가 없음
    • 프로그래밍 복잡도 다소 낮아짐

1980년대

  • 1980년대 이전
    • 대형 컴퓨터를 여러 명이 접속해서 사용 (UNIX)
  • 1980년대 이후
    • Personal Computer (PC)
    • CLI에서 GUI로 변화.
    • 80년대 초반: CLI 환경
    • 89년대 중후반: GUI 환경 (마우스 사용)

CLI(Command Line Interface): 터미널 환경
GUI(graphical User Interface): GUI 환경

1990년대

  • GUI 환경과 개인용 컴퓨터로 인해 사용자가 증대되어 응용프로그램 시대가 도래
    • 엑셀, 워드 프로세서 등등
    • Windows OS 대중화
  • 네트워크 기술 발전
    • 월드와이드웹 (WWW) 인터넷 대중화
  • 오픈 소스 운동 활성화 시작
    • UNIX 계열 OS + 응용 프로그램 자체 개발, 소스 오픈
    • LINUX(리눅스) 운영체제가 만들어지게 되며 LINUX(리눅스)는 소스가 오픈되어 있으며 무료

Killer Application
해당 기기에서만 플레이할 수 있는 어플리케이션
ex1 플레이스테이션에서만 플레이 가능한 위닝 게임
ex2 Windows OS에서만 사용 가능한 엑셀 프로그램

2000년대 이후

  • 오픈 소스 활성화
    • LINUX(리눅스) 운영 체제
    • Apache(아파치, 웹서버)
    • MySQL(데이터베이스)
    • 안드로이드, 딥러닝, 데이터 사이언스, IoT 관련
  • 가상 머신, 대용량 병렬 처리등 활성화
